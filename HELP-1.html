Verdure
World Editor

A browser-based tool for building the maps, entities, portals, dialogs and interactions that make up the Verdure game world. Everything you create is stored in a single world.json file you can deploy alongside the game engine.

🌱 Version 0.1 · Single-file editor · No server required

01 — OverviewWhat is the World Editor?

The Verdure World Editor is a standalone HTML file (editor/index.html) that runs entirely in your browser. It has three tabs, each serving a distinct purpose:

🗺
Maps Tab
Create and edit maps. Upload backgrounds, place objects, add portals, set climate zones and blight levels.
🌾
Entities Tab
Define the library of things that exist in the world — tools, plants, buildings, creatures, hazards, portals.
{ }
JSON Tab
Preview, copy, or download the complete world.json data file for use by the game engine.

All data is auto-saved to your browser's localStorage as you work. Use Export JSON to download a permanent copy, and Import to load a previously exported file.

💡
The editor and game engine share the same world.json schema. Define your entities and maps here, export, and the game engine reads the result directly. No conversion step needed.

03 — Quick StartBuilding Your First Scene

  1. Create some entities first Switch to the Entities tab and click + New Entity. Create at least one tool (e.g. Trowel), one plant (e.g. Wildflower Seed), and one hazard (e.g. Litter Pile). Give each a name, category, and optionally upload a sprite image.
  2. Create a map Switch to the Maps tab and click + New Map. Name it, choose Top-down type, pick a climate zone, and set a blight level (75 = heavily neglected). Click Create Map.
  3. Upload a background image With your map selected, click the 🖼 button in the canvas toolbar and upload an image. AI-generated images work well — try prompting for a "top-down view of an abandoned urban lot, hand-drawn illustration style."
  4. Place objects Select the ＋ Place tool, then click anywhere on the canvas. A modal appears — choose an entity from the dropdown and click Place. Repeat for each object. Use the ↖ Select tool to click and drag objects to reposition them.
  5. Add a portal to another map Create a second map (e.g. a Garden Shed, side-view type). Back on the first map, select the 🚪 Portal tool and click where the door should be. Set the target map in the modal. Click Add Portal.
  6. Export your world Click Export JSON in the header. Save world.json into your project's data/ folder alongside the game engine.

04 — Maps TabWorking with Maps

Maps are the spaces players inhabit. Each map has a background image, a set of placed objects, portals connecting to other maps, and metadata like climate zone and blight level.

Creating a Map

Click + New Map in the left panel. The modal asks for:

FieldDescription
Map NameDisplay name, e.g. "Abandoned Corner Lot" or "Rooftop Garden"
TypeTop-down for outdoor/overhead spaces; Side-view for interiors like sheds, rooms, underground
Climate Zone1–12, matching US Hardiness Zones. Controls which plants can grow here
Blight Level0–100. How neglected the space is. 0 = fully restored, 100 = completely blighted
DescriptionFlavour text shown to the player when they enter

Uploading a Background Image

With a map selected, click the 🖼 button in the canvas toolbar. Choose any image file — it's stored as base64 inside world.json. Click ✕ to remove a background.

🎨
For a consistent hand-drawn aesthetic, generate backgrounds using an AI image tool. Suggested prompts:
  • "top-down view of an abandoned urban lot with cracked concrete, hand-drawn illustration, muted colours"
  • "side-view interior of a dusty garden shed, flat 2D illustration, warm earthy tones"
  • "top-down rooftop garden, sketch style, overgrown planter boxes"

Canvas Toolbar Tools

ToolKeyWhat it does
↖ SelectDefaultClick to select a placed object; drag to reposition it. Click empty canvas to deselect.
＋ Place—Click anywhere on the canvas to open the Place Object modal. Choose an entity from your library.
🚪 Portal—Click anywhere to add a portal at that position. Opens portal settings modal.
🖼 Background—Upload a background image for the current map.
✕ Clear BG—Remove the current background image.

Map Properties Panel

When no object is selected, the right panel shows map-level properties. Edit any field and click Apply to save. The climate zone grid lets you click any zone number to change it.

Portals

Portals are doors, gates, paths, ladders — any transition point between maps. There are two ways to create them:

Method 1 — Portal Tool (Maps tab)

  1. Select the 🚪 Portal tool
  2. Click the canvas where you want the portal to appear
  3. Fill in the label (e.g. "Shed Door"), target map, and required level
  4. Click Add Portal

Method 2 — Entity with category "portal" (Entities tab)

  1. Create an entity with category set to portal
  2. Add a sprite (e.g. a drawn door, gate, or ladder)
  3. Switch to Maps tab, use the ＋ Place tool, select your portal entity
  4. When a portal-category entity is selected, additional fields appear for target map and required level
ℹ
Both methods produce identical data in world.json. The entity method is preferred when you want a custom sprite for the portal. The tool method is faster for quick, iconographic portals.

Editing a Portal After Placement

Select the portal object on the canvas using the Select tool. The properties panel shows a Portal Target Map dropdown — change it and click Apply to relink. An ⚠ warning appears if the portal has no target set.

Deleting a Portal

Select the portal object and click Delete in the properties footer. This removes both the object and its corresponding portal record from map.portals[].

05 — Entities TabBuilding Your Entity Library

Entities are the reusable definitions of everything that can exist on a map — tools, plants, buildings, creatures, hazards, portals, decorations, and hidden objects. You define them once here, then place instances of them on maps.

Use the category filter tags at the top of the left panel to filter the list by type. Click + New Entity to create one.

Identity & Sprite

FieldDescription
NameDisplay name shown to players and in the editor
CategoryOne of 8 types — see Entity Categories
DescriptionFlavour text describing the entity
Raritycommon, uncommon, rare, or legendary
Sprite / ImageThe visual used on the map canvas and in-game

Setting a Sprite

In the Identity section of the entity form:

  • Click the 📁 Upload Image File button — pick any PNG, JPG, GIF, or WebP from your computer. It's converted to base64 and stored immediately.
  • Or paste a URL into the text field below the button (e.g. from a hosted assets folder).
  • The thumbnail on the left previews the image. Click it to re-upload.
  • Click ✕ Remove Sprite to clear it.
🎨
Use transparent-background PNG files for sprites placed on map backgrounds. Generate them with AI tools using prompts like "isometric pixel art watering can, transparent background, 128x128".

Properties

FieldDescription
Zone RequirementsWhich climate zones (1–12) this entity can appear in. Leave all off = available everywhere.
EffectPassive gameplay effect this entity has when placed. See Effects Reference.
Grow Time (hours)For plants: how many real-world hours before it reaches the next growth state.
Growth Speed MultiplierSlider from 0.1× to 5×. Multiplies grow time at runtime. 2.0 = grows twice as fast.
CollectibleCan the player pick this up and add it to their inventory?
InteractableCan the player interact with this entity?
HiddenMust be discovered before it becomes visible. Used for hidden object gameplay.
TradeableCan this entity be traded between players?
RespawnsDoes this entity reappear after being collected or harvested?
Respawn AfterVisible when Respawns is checked. Number of hours before it reappears.

Sounds

Ambient Sound (entity-level)

A looping audio file that plays when a player is near this entity. Set it in the Properties section:

  • Click 🔊 Upload Audio to upload an MP3, OGG, or WAV file
  • Or paste a URL into the text field
  • Click ▶ to preview it in the browser
  • Click ✕ to remove it

Interaction Sounds (per interaction)

Each interaction can have its own one-shot sound that plays when triggered. See the 🔔 Sound row in each interaction — upload a file or paste a URL.

States

States represent the different conditions an entity can be in. For example, a plant might have states: seed, sprout, bloom. A building might have ruined, restoring, restored.

Each state has an ID (used in code) and a Label (shown to players). Click + Add to add a new state. The first state is used as the default when an instance is placed.

💡
Always include an idle state as the starting point. The game engine transitions between states based on interactions and grow timers.

Required Tools to Place/Plant

Use the Required Tools to Place/Plant section to gate who can place this entity. Click + Add and select a tool entity from the dropdown. Players must have all listed items in their inventory to place this entity on a map.

For example: a rare plant might require both a Trowel and a pair of Gardening Gloves to plant.

This is stored as place_requires: ["entity_id_1", "entity_id_2"] on the entity.

Interactions

Interactions define what happens when a player does something to this entity. Click + Add to add one. Each interaction has:

FieldDescription
TriggerWhat the player does: click, collect, water, harvest, inspect, plant
ActionWhat happens: collect, grow, reveal, unlock, reward, transform, gift, plant, harvest, dialog
RewardWhat the player receives, e.g. seed_packet, 10_coins, trowel
🔧 RequiresOptional. A tool entity the player must have in their inventory for this interaction to fire.
🔔 SoundOptional. Audio file or URL to play when this interaction fires.
🔍
The difference between place_requires (entity level) and 🔧 Requires (interaction level): place_requires gates whether the entity can be planted at all; Requires gates whether a specific action can be performed on it once placed.

Dialog Trees

To add a conversation to an entity, set an interaction's Action to dialog. An ✎ Edit Dialog button appears. Click it to open the dialog tree editor.

Dialog Editor Overview

The dialog editor is a full node-graph canvas. Each node represents a moment in conversation:

ElementDescription
NodeA single conversational beat. Has a speaker, text, and 0–4 player choices.
Start NodeHighlighted in amber — this is the first node shown when dialog begins. Click ⚑ on any node to set it as start.
ChoiceA player response option. Links to another node (branching) or null (ends conversation).
Connection LineGreen/amber bezier curves show which choices lead to which nodes.

Building a Dialog Tree

  1. Add nodes — click + Node in the toolbar. A new node appears on the canvas.
  2. Position nodes — drag them around the canvas to lay out the conversation flow. Use Auto Layout to arrange them in a grid automatically.
  3. Edit a node — click any node to open its editor on the right. Set the Speaker name and dialog Text.
  4. Add choices — click + Add in the Choices section. Enter the choice text and select which node it leads to. Optionally add a reward for choosing it.
  5. Link choices — in the "leads to" dropdown, select the destination node. If left as "end conversation", clicking that choice ends the dialog.
  6. Set start node — click ⚑ on the node that should open the conversation. It turns amber.
  7. Click Done — saves the dialog and returns to the entity form.
💡
Node IDs are editable — give them meaningful names like greeting, accepts_trade, refuses_trade to make the graph readable. Renaming a node automatically updates all references to it.

Dialogs are stored in world.dialogs[] and referenced from the interaction via dialog_id.

06 — Placed ObjectsPlacing & Configuring Objects

When you place an entity on a map, you create an instance of it. The instance inherits its entity's properties but can have its own position, size, rotation, animation, state, and label.

Placing an Object

  1. Select the ＋ Place tool
  2. Click anywhere on the canvas
  3. In the modal, choose an entity from the dropdown — if you choose a portal category entity, additional portal fields appear
  4. Optionally override the label, then click Place

Selecting & Moving

  • Switch to the ↖ Select tool
  • Click an object to select it — a green outline appears and the properties panel updates
  • Drag the selected object to reposition it
  • Click empty canvas space to deselect

Size, Z-Order & Rotation

PropertyRangeDescription
Size16–256 pxHow large the object appears on the canvas. Slider updates live.
Z-Order1–100Depth layer. Higher values render on top of lower ones. Use this to layer objects correctly — a tree (z:5) behind a character (z:50).
Rotation0–360°Rotates the sprite. Useful for paths, directional objects, fallen items. Slider updates live.

Animation

Select an animation style by clicking one of the tag buttons in the properties panel. The animation applies immediately on the canvas.

🌿 sway Gentle left-right rotation. Plants, trees, long grass.
🐛 jitter Rapid small shake. Creatures, hazards, active machines.
⭐ pop Slow scale pulse. Collectibles, rewards, highlighted items.
✨ float Gentle up-down bob. Magical items, seeds, ethereal objects.
💫 glow Opacity and brightness pulse. Hidden objects, rare finds.
🪨 none Static. Buildings, fixed structures, background elements.

Animations are pure CSS keyframes. The game engine applies the same class names (anim-sway, anim-jitter, etc.) to replicate the effect.

Collision & Trigger Radius

PropertyOptions / RangeDescription
Collisionwalkable / solid / interact_onlywalkable: player passes through freely. solid: blocks player movement. interact_only: no physical blocking but interaction is available when in range.
Trigger Radius1–30%How close the player must be (as a % of map width) to trigger this object's interactions. Smaller = must be right next to it. Larger = activates from further away.

07 — JSON TabPreviewing World Data

The JSON tab shows a live preview of your complete world.json. Base64 image and audio data is replaced with a placeholder in the preview ([base64 image — included in export]) to keep it readable, but the actual exported file contains all data.

Toolbar

  • Copy — copies the full JSON (with base64) to your clipboard
  • Download — downloads as world.json

08 — Schemaworld.json Structure

The complete world is a single JSON object with four top-level keys:

"meta": { title, version, created } // world metadata
"maps": [ Map, ... ] // all map definitions
"entities": [ Entity, ... ] // entity library
"dialogs": [ Dialog, ... ] // dialog trees (created on demand)

Map Object

{
  "id": "map_1",
  "name": "Abandoned Corner Lot",
  "type": "top-down", // or "side-view"
  "zone": 6,
  "blight_level": 80, // 0–100
  "description": "...",
  "background": "data:image/png;base64,...", // or null
  "portals": [ { id, to, label, requires_level, x, y } ],
  "objects": [ PlacedObject, ... ]
}

PlacedObject

{
  "id": "obj_1",
  "entity_id": "entity_3", // references entity library
  "label": "", // override display name, or blank
  "x": 0.35, // 0.0–1.0 (fraction of map width)
  "y": 0.6, // 0.0–1.0 (fraction of map height)
  "size": 48, // pixels
  "z_order": 10,
  "rotation": 0, // degrees
  "animation": "sway", // none|sway|jitter|pop|float|glow
  "collision": "walkable", // walkable|solid|interact_only
  "trigger_radius": 8, // % of map width
  "state": "idle",
  "notes": ""
  // portal objects also have: type, portal_id
}

Entity Object

{
  "id": "entity_1",
  "name": "Rusty Watering Can",
  "category": "tool",
  "rarity": "common",
  "description": "...",
  "sprite": "data:image/png;base64,...",
  "zone_requirements": [3, 4, 5], // [] = any zone
  "effect": "speeds_growth",
  "grow_time_hours": 24,
  "growth_speed_multiplier": 1.0,
  "collectible": true,
  "interactable": true,
  "hidden": false,
  "tradeable": true,
  "respawn": true,
  "respawn_hours": 24,
  "ambient_sound": "data:audio/mp3;base64,...",
  "place_requires": ["entity_2"],
  "states": [{ id, label }, ...],
  "interactions": [ Interaction, ... ]
}

Interaction Object

{
  "trigger": "harvest",
  "action": "reward",
  "reward": "tomato",
  "requires_entity_id": "entity_2", // tool needed
  "sound": "data:audio/mp3;base64,...",
  "dialog_id": "dlg_001" // if action is "dialog"
}

Dialog Object

{
  "id": "dlg_001",
  "name": "Gardener dialog",
  "start_node": "node_start",
  "nodes": [
    {
      "id": "node_start",
      "speaker": "Old Gardener",
      "text": "You found my trowel!",
      "x": 40, "y": 40, // canvas position

        { "id": "c1", "text": "Here you go", "next": "node_thanks", "reward": "" },
        { "id": "c2", "text": "What's it worth?", "next": "node_trade", "reward": "" }
      ]
    }
  ]
}

09 — Export & ImportSaving and Loading Your World

Auto-Save

Every change is automatically saved to your browser's localStorage. This persists across page refreshes and browser restarts — but only in the same browser on the same device.

⚠
localStorage can be cleared by browser settings, private browsing sessions, or clearing site data. Always export a world.json file after significant work.

Exporting

Click Export JSON in the header at any time. This downloads world.json with all data including full base64 images and audio. Place this file in your project's data/ folder for the game engine to read.

Importing

Click Import in the header and select a world.json file. This replaces all current data. The file must have both maps and entities arrays at the top level.

Deploying to GitHub Pages

  1. Place editor/index.html in your repository
  2. Place your exported world.json in data/world.json
  3. Enable GitHub Pages from the repository settings (source: main branch, root or /docs)
  4. The editor is accessible at https://yourusername.github.io/yourrepo/editor/

11 — ReferenceEffects

The Effect field on an entity defines its passive impact on the game world when placed. The game engine reads this to calculate gameplay outcomes.

EffectDescription
speeds_growthReduces grow time of nearby plants
slows_blightReduces the rate at which the map's blight level increases
restores_soilActively reduces blight level over time
attracts_creaturesIncreases chance of creature entities spawning nearby
provides_shelterAllows cold-zone plants to survive in adjacent areas
reveals_hiddenMakes hidden_object entities in trigger radius discoverable
blocks_hazardPrevents hazards from spreading into adjacent areas
generates_seedsPeriodically produces seed items for the player to collect

12 — ReferenceClimate Zones

Verdure uses the 12 USDA Plant Hardiness Zones as its geography system. Each map has a zone and each entity can restrict which zones it's available in.

ZoneClimateExamples
1–2Subarctic / AlpineNorthern Canada, Alaska high elevations
3–4Cold continentalMontana, Minnesota, Northern New England
5–6TemperateChicago, Denver, most of the Midwest
7–8Mild temperatePacific Northwest, Virginia, Tennessee
9–10Subtropical / MediterraneanLos Angeles, Phoenix, Gulf Coast
11–12TropicalSouth Florida, Hawaii, Puerto Rico

Zone restrictions on entities create natural asymmetry — a Zone 9 player can grow a Saguaro Cactus that a Zone 3 player cannot. This incentivises trading between players in different regions.

13 — ReferenceRecommended Workflow

For efficient world-building, follow this order:

  1. Define your entity library first — You can't place entities that don't exist yet. Build out tools, plants, hazards, portals, and any hidden objects before working on maps.
  2. Create maps in logical order — Start with the "entry" map a player first sees. Then create connected maps (rooms, rooftops, sheds) and link them via portals.
  3. Upload backgrounds before placing objects — It's much easier to position objects accurately when you can see where they should sit in the scene.
  4. Use blight level intentionally — High blight (75–100) = lots of hazards to clear, few plants possible. Low blight (0–25) = a flourishing space. Set this to match the visual state of your background image.
  5. Add interactions and dialogs last — Get the world looking right first. Layer in gameplay mechanics (interactions, dialog trees, required tools) once the visual layout is solid.
  6. Export frequently — localStorage is convenient but not permanent. Export a world.json at the end of every working session.
  7. Version your JSON — Commit world.json to your GitHub repository after each significant session. This gives you a full history and makes it easy to roll back mistakes.
🌱
The editor is designed to grow alongside your game. Start with a small, complete world — two or three connected maps with a handful of entities — and expand it incrementally. A focused small world is more fun to design than an incomplete large one.
