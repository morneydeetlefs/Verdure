Verdure
World Editor

Browser-based tool for building the maps, entities, portals, masks, dialogs and interactions that make up the Verdure game world. Everything exports to a single world.json file read by the game engine.

🌱 Version 0.3 · Editor + Game Engine · Mini-Games Support

01What is Verdure?

Verdure is a browser-based guerrilla gardening game where players reclaim neglected spaces — clearing litter, planting seeds, restoring buildings — without needing to physically move. The project consists of two HTML files:

🛠
editor/index.html
World Editor — create maps, define entities, draw masks, build dialog trees, configure mini-games, export world.json.
🎮
game/index.html
Game Engine — loads world.json, renders maps, handles player interactions, mini-games, inventory, XP, grow timers.
📄
data/world.json
The exported world — all maps, entities, dialogs, masks. Shared between editor and engine.

04Quick Start

  1. Create entities first — Switch to the Entities tab. Create at least: one hazard (Litter Pile), one tool (Trowel), one plant (Wildflower Seed), one portal (Shed Door), one mini-game (Puzzle Box). Give each a name, category, and upload a sprite if you have one. For the mini-game, set the game HTML path and configure rewards.
  2. Create maps — Switch to Maps tab. Click + New Map. Name it, set type to Top-down, choose climate zone 6, blight level 75. Create a second map (Garden Shed, Side-view).
  3. Upload backgrounds — Select your first map, click 🖼 in the toolbar, upload a generated background image.
  4. Draw masks — Select the ⬡ Mask tool. Click points around non-plantable areas (walls, paths, sky). Double-click to close the polygon. Choose No Plant Zone.
  5. Place objects — Select ＋ Place tool, click canvas to open the placement modal, choose an entity. Use ↖ Select to drag objects to position them.
  6. Link maps with a portal — Select 🚪 Portal tool, click where the door goes, set target to your second map.
  7. Export — Click Export JSON. Save world.json into data/world.json.
  8. Play — Open game/index.html, load your world.json, enter your name and start playing.

05Working with Maps

Creating a Map

Field Description
Map Name Display name shown in the minimap and on-screen banner
Type Top-down — outdoor overhead view. Side-view — interior flat perspective
Climate Zone 1–12 matching USDA Hardiness Zones. Controls which plants can grow here
Blight Level 0–100. How neglected this space is. 0 = restored, 100 = abandoned wasteland

Background Images

Click 🖼 in the canvas toolbar to upload a background. Images are stored as base64 inside world.json. The canvas maintains a 16:9 aspect ratio and fits the available editor width — backgrounds display with letterboxing (object-fit: contain) to prevent clipping across screen orientations.

🎨
Suggested AI image prompts for backgrounds:
  • "top-down view of an abandoned urban lot, cracked concrete, weeds, hand-drawn illustration style, muted palette"
  • "side-view interior garden shed, dusty shelves, tools hanging, flat 2D illustration, warm earthy tones"
  • "top-down rooftop garden, overgrown planter boxes, city skyline, sketch style"
  • "top-down community park, neglected flowerbeds, broken benches, watercolour style"

Portals

Portals are transition points between maps — doors, gates, paths, ladders. Two creation methods produce identical JSON:

Method A — Portal Tool

  1. Select 🚪 Portal tool
  2. Click where the portal should appear on the canvas
  3. Enter a label, choose the target map, set required level
  4. Click Add Portal

Method B — Portal Entity

  1. Create an entity with category portal in the Entities tab (add a door/gate sprite)
  2. Use the ＋ Place tool and select your portal entity
  3. Extra portal fields appear — set target map and required level

To edit a portal after placement: select it with the Select tool — a Portal Target Map dropdown and Requires Level field appear in the properties panel. Click Apply to save.

ℹ
Portals show ⚠ in the properties panel if their target is unlinked. Deleting a portal object also removes its entry from map.portals[] automatically.

Polygon Masks

Masks define zones with special rules — areas where planting is blocked, movement is restricted, or only certain interactions apply.

Drawing a Mask

  1. Select the ⬡ Mask tool from the canvas toolbar. A hint banner appears at the bottom of the canvas.
  2. Click to place points. Each click adds a vertex of the polygon. A yellow dashed preview shows the shape being built.
  3. Double-click to close the polygon (minimum 3 points). A modal asks for the mask type and an optional label.
  4. Choose a mask type and click Save Mask.

Mask Types

Type Colour Effect in Game
no_plant Red Player cannot plant entities here
no_walk Blue Blocks player movement
interact_only Amber Player can interact but not plant
plant_only Green Designated planting bed
water Cyan Water surface — special logic

Select any mask with the Select tool to edit its type, label, or reshape it by dragging the white vertex handles. Click Delete in the properties footer to remove it.

06Entity Library

Entities are reusable definitions — you create them once, then place instances on maps. The entity stores all properties; the placed instance stores only position, size, rotation, animation, state and overrides.

Identity & Sprite

Field Description
Name Shown to players and in editor lists
Category One of 8 types — see Categories Reference
Description Flavour text shown in the action popup
Rarity common, uncommon, rare, legendary
Sprite / Image PNG/JPG used on the canvas and in-game. Upload or paste URL.
💡
Use transparent-background PNG sprites. AI prompts: "isometric pixel art rusty trowel, transparent background, 128×128" or "flat illustration wildflower seed packet, no background, clean edges".

Properties

Field Description
Zone Requirements Which climate zones this entity appears in. Leave all off = available everywhere.
Effect Passive gameplay effect when placed. See Effects Reference.
Grow Time (hours) Real-world hours per growth stage transition.
Growth Speed Multiplier 0.1× (slow) to 5× (fast). Multiplies grow time at runtime.
Collectible Can player pick this up into inventory?
Interactable Can player interact with this entity?
Hidden Invisible until revealed via an inspect → reveal interaction.
Tradeable Can be traded between players (future social feature).
Respawns Reappears after being collected or harvested.
Respawn After (hours) How long before it reappears. Shown when Respawns is checked.

Sounds

Ambient Sound — loops while player is on a map containing this entity. Upload MP3/OGG/WAV or paste a URL. Click ▶ to preview. Stored as ambient_sound on the entity.

Interaction Sound — one-shot, plays when a specific interaction fires. Set per-interaction via the 🔔 Sound row. Upload or URL.

States

States represent the lifecycle of an entity. Each has an ID (code) and Label (player-facing). Examples:

Entity type States
Plant seed → sprout → flower → bloom
Building ruined → restoring → restored
Tool (hidden) idle → found
Hazard present → cleared

Click + Add to add states. The first state is always the default when placed.

Required Tools to Place/Plant

In the Required Tools to Place/Plant section, click + Add and select tool entities the player must carry. All listed tools must be in inventory to plant this entity. Stored as place_requires: ["entity_id"].

Example: a rare Desert Rose might require both a Trowel and a Watering Can to plant.

Interactions

Each interaction has: Trigger (what the player does) → Action (what happens) → Reward + optional 🔧 Requires (tool needed) + optional 🔔 Sound.

Trigger Action Typical use
click collect Simple pickup — tool, seed, item
click dialog Start a conversation tree
inspect reveal Uncover a hidden object
water grow Advance a plant to next state
harvest reward Collect output from a fully grown plant
click transform Change building state (restore)
click unlock Open a locked area or container
plant grow Planting a seed from inventory
play play_minigame Launch a mini-game popup
🔍
place_requires vs 🔧 Requires: place_requires gates whether an entity can be planted at all. requires_entity_id per-interaction gates whether a specific action can be performed on it once placed.

Mini-Game Settings

For entities in the minigame category, additional settings appear to configure the mini-game experience.

Setting Description Example
game_html Path to the HTML file containing the mini-game code. Should be relative to the game root. cc-game.html
totalMovesAllowed Maximum number of moves the player can make in the mini-game before it ends. 30
rewards Array of rewards for completing the mini-game. Supports items, coins, energy, and XP. ["10_coins", "watering_can", "5_energy", "10_xp"]

The mini-game HTML file should implement a startMergeMania(config) function that accepts a config object with totalMovesAllowed and posts messages back to the parent window with results like {type: 'minigame_complete', score: 1500}.

🎮
Mini-Game Integration: When a player interacts with a mini-game entity, it opens the specified HTML in a popup window. Upon completion, rewards are automatically granted based on the configured rewards array.

Dialog Trees

Set an interaction's Action to dialog → click ✎ Edit Dialog → the full-screen node graph editor opens.

Using the Dialog Editor

  1. Add nodes with + Node. Each node is one moment of conversation: a speaker, text, and player choices.
  2. Position nodes by dragging them. Use Auto Layout to arrange in a grid automatically.
  3. Click a node to open its editor on the right. Set Speaker and Text.
  4. Add choices — each choice links to another node or ends the conversation. Optionally add a reward per choice.
  5. Set the start node — click ⚑ on the first node. It turns amber.
  6. Click Done to save and return.
💡
Give nodes meaningful IDs like greeting, trade_accepted, refuses_help — renaming a node automatically updates all references to it.

Size, Z-Order & Rotation

Property Range Description
Size 16–256 px Sprite display size. Slider updates live on canvas.
Z-Order 1–100 Depth layer. Higher = renders on top. Building (z:5), plant (z:10), character (z:50).
Rotation 0–360° Rotates the sprite. Rotation is independent of animation — they do not interfere.

Animation

Click a tag to apply live. Animation uses an inner wrapper element separate from the outer rotation transform, so rotation and animation always work together correctly.

🌿swayGentle rotation. Plants, tall grass, flags.
🐛jitterRapid shake. Creatures, hazards, machines.
⭐popScale pulse. Collectibles, rewards.
✨floatUp-down bob. Magical items, seeds.
💫glowBrightness pulse. Hidden or rare objects.
🪨noneStatic. Buildings, fixed structures.

Collision & Trigger Radius

Property Options Description
Collision walkable / solid / interact_only walkable: pass through. solid: blocks movement. interact_only: no block but interactable in range.
Trigger Radius 1–30% of map width How close the player must be to trigger interactions.

08Game Engine Overview

The game engine (game/index.html) is a self-contained point-and-click experience. Players click on objects to interact — no movement required. Everything is driven by the world.json exported from the editor.

🗺
Map Rendering
Background, objects at correct position/size/rotation, animations, portal transitions with fade.
👆
Interactions
Click objects to open action popup. Tool requirements enforced. All 9 action types handled.
�
Mini-Games
Popup mini-games with configurable rewards. Supports external HTML games with message passing.
�🎒
Player & Inventory
Level, XP, energy pips, scrollable inventory with sprites. Persisted to localStorage.
⏱
Grow Timers
Real-time growth, respawn timers, countdown badges on growing objects.

Loading a World

The engine tries to auto-load world.json from ../data/world.json, then world.json, then data/world.json. Auto-load only works on http/https. When running from a local file:// URL, it skips straight to the file picker — click 📂 Open world.json and select your exported file.

💡
The fastest local workflow: install the VS Code Live Server extension. Right-click game/index.html → Open with Live Server. Auto-load will work and the engine reloads when you re-export.

Player & HUD

On first launch a name setup screen appears. Player data is stored in localStorage as verdure_player — separate from world data so exporting never affects progress.

HUD Element Description
Player Name + Level Current level, XP toward next level, XP progress bar
Energy Pips 20 pips. Each action costs 1–2 energy. Regenerates 1 pip per 3 minutes.
Inventory Scrollable slots showing collected entities with sprite and quantity. Click to select/use.
Map Name + Blight Bar Current map, climate zone, type, and blight level visualised as a red bar.
🗺 Maps button Opens minimap panel — lists all maps. Visited maps are navigable directly.
⚙ Settings Change name, master volume, label toggle, reset progress.

Interactions

Click any object on the map. A popup appears above it showing available actions. Actions with unmet tool requirements show a 🔒 icon and are disabled. The entity's current state is shown at the top of the popup.

Action Energy cost What happens
collect 1 Removes object from map, adds to inventory, gives XP
grow / water 1 Advances entity to next state, starts grow timer
harvest 2 Only works at final state. Gives reward, resets if respawning.
reveal (inspect) 1 Makes hidden entity visible with a flash animation
transform 0 Advances building/object state
dialog 0 Opens the dialog box with branching conversation
reward 0 Gives reward item directly
plant 2 Advances a planted item's state (from inventory plant mode)
play_minigame 0 Opens mini-game in popup window, grants configured rewards on completion

Energy System

Players start with 20 energy pips. Most actions cost 1 energy, harvesting costs 2. When energy runs out, a toast shows the time until the next pip refills (1 pip every 3 minutes). Energy regenerates whether the game is open or closed — it calculates elapsed time when you return.

Planting from Inventory

Tap a plant-category item in your inventory. A green "Plant mode" banner appears at the top of the screen and the cursor changes to a crosshair. Click anywhere on the map to plant. The engine checks:

  1. Does the player have all place_requires tools?
  2. Is there enough energy?
  3. Is the click position inside a no_plant mask zone?

If all checks pass, the object is planted at that position, removed from inventory, and added to the map with a 🌱 particle effect.

Grow Timers

Growing objects show a countdown badge (2h 30m or Ready!). The engine ticks every 10 seconds and automatically advances state when the grow time has elapsed. The growth_speed_multiplier on the entity scales the time — 2.0 halves the grow time, 0.5 doubles it.

Dialog in the Game

When a dialog interaction fires, the dialog box slides up from the bottom of the screen. The speaker name appears in green, the text in italic. Player choices appear as buttons. Clicking a choice can lead to another node, give a reward, or end the conversation.

11Export & Import

Auto-Save

Every change saves automatically to browser localStorage. This persists across refreshes in the same browser but can be lost if site data is cleared.

⚠
Always export a world.json after significant work. localStorage is a convenience cache, not a permanent save.

Exporting

Click Export JSON in the editor header. All base64 image and audio data is included. Place in data/world.json for the game engine to auto-load on http/https.

Importing

Click Import and select a world.json file. This replaces all current data. The file must have both maps and entities arrays.

GitHub Pages Deployment

  1. Push your project folder to a GitHub repository
  2. Enable GitHub Pages (Settings → Pages → Branch: main, folder: root)
  3. Place data/world.json in the repository
  4. Editor: https://user.github.io/repo/editor/
  5. Game: https://user.github.io/repo/game/

13Rewards Reference

Rewards are granted when players complete interactions or mini-games. The giveReward() function handles all reward types automatically.

Reward Format Description Example
item_name Adds the specified entity to inventory watering_can, wildflower_seed
N_coins Adds N coins to player currency 10_coins, 50_coins
N_energy Restores N energy pips to player 5_energy, 10_energy
N_xp Grants N experience points toward leveling 25_xp, 100_xp
💰
Reward Processing: Rewards are processed in order. Multiple rewards can be granted simultaneously (e.g., ["10_coins", "watering_can", "5_energy"]). Invalid reward formats are ignored with a console warning.

14Effects Reference

Effect Description
speeds_growth Reduces grow time of nearby plants
slows_blight Reduces the rate at which blight level increases
restores_soil Actively reduces blight level over time

16Recommended Workflow

  1. Build entity library first. You cannot place what doesn't exist yet. Define all tools, plants, hazards, portals, and NPCs before working on maps.
  2. Create maps in logical order. Start with the entry map. Then create rooms/areas connected by portals.
  3. Upload backgrounds before placing objects. Positioning is much easier when you can see the scene.
  4. Draw masks after backgrounds. Trace non-plantable areas (walls, paths, water) with the ⬡ Mask tool.
  5. Place objects with z-order in mind. Background decorations at z:3, ground-level objects at z:10, foreground details at z:20+.
  6. Add interactions and dialogs last. Get the world looking right before layering in gameplay logic.
  7. Export frequently. Export a world.json at the end of every session. Commit it to GitHub after each significant milestone.
  8. Test in the game engine. Open game/index.html, load your world, walk through every portal, trigger every interaction, check grow timers fire correctly.
